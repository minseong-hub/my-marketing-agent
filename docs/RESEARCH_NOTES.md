## Phase 1 — 발견 사항 (자체 개발 가능 = 🟢 / 사용자 결정 = 🟡 / 외부 의존 = 🔴)

### 🟢 즉시 개선 가능한 영역 (Phase 2에서 작업)

#### R-1. 비서 텍스트 응답 보관함 미저장 + 300자 잘림
**위치**: `lib/agents/runner.ts:294-298`
**문제**: `response.stop_reason === "end_turn" && toolUses.length === 0`일 때 `textContent.slice(0, 300)`으로 success 로그만 emit. 비서가 도구 사용 없이 마크다운 보고서로 답한 경우 본문이 잘리고, 보관함에도 저장되지 않음 — 사용자가 결과를 다시 찾으려면 못 찾음.
**해결**: 풀 텍스트를 `library_items`에 자동 저장 (kind: `agent_response`), 로그는 요약 + "보관함에서 보기" 링크.

#### R-2. 토큰 사용량/비용 추적 부재
**위치**: `lib/agents/runner.ts:107-113` (response 받은 직후)
**문제**: `response.usage.input_tokens / output_tokens / cache_*`를 무시. 호출당 비용 측정 불가, 한도 강제(quota.ts)는 세션 수만 카운팅 — 한 세션에서 수십 턴 도는 경우 실 토큰은 한도의 수배일 수 있음.
**해결**: `claude_token_usage` 테이블 추가 + 매 응답마다 `db.recordTokenUsage(userId, agentType, sessionId, usage)` 기록 + automation/summary에 비용 추정치 노출.

#### R-3. Prompt Caching 미적용
**위치**: `lib/agents/runner.ts:107`
**문제**: 시스템 프롬프트(2~3k 토큰) + 도구 정의(1k 토큰)가 매 턴 풀 전송. Claude API의 `cache_control` 사용 시 80% 입력 비용 절감 가능.
**해결**: system prompt와 tools 마지막 블록에 `cache_control: { type: "ephemeral" }` 추가 → 5분 TTL 캐시.

#### R-4. 작업 중단(abort) 미지원
**문제**: 사용자가 30 iteration 도는 비서를 멈출 수 없음. 예산 초과 위험.
**해결**: `db.agent_sessions.cancel_requested` 컬럼 + runner 루프 매 턴 폴링 + UI에서 ⏹ 정지 버튼.

#### R-5. Manual Ops 비서당 2개 → 5~7개로 부족
**위치**: `components/agents/AssistantDetail.tsx` CREW_MAP.manualOps
**문제**: 스펙은 비서당 5~7개, 현재 2개씩 8개. 셀러 자주 쓰는 작업 미커버.
**해결**: 비서당 5개로 확장. 추천 추가 작업:
- 마키: 인스타 릴스 시나리오 / 부정 리뷰 답글 / 인플루언서 협업 DM
- 데일리: 옵션 설명 카피 / 배송·환불 안내 / 리뷰 응답 템플릿
- 애디: 부정 키워드 추출 / 비효율 광고 일시정지 후보 / 광고 소재 5종
- 페니: 부가세 시뮬 (페니 계산기와 연계) / 환불 정책 카피 / 단순 손익 코멘트

#### R-6. AssistantDetail 통계 게이지 가짜 (하드코딩)
**위치**: `components/agents/AssistantDetail.tsx:40-45 등`
**문제**: 모든 사용자가 "SNS 참여율 92%" 동일하게 표시 → 신뢰도 ↓
**해결**: 라벨에 "예시 성과 — 실 데이터 연동 시 변경" 명시 추가 또는 사용자 실 데이터 부재 시 안내 카드로 교체.

#### R-7. 빈 상태 UI 약함
**문제**: `/app/library`, `/app/products`, `/app/brand` 모두 텍스트만. 첫 사용자에게 "어디서 시작?" 안내 부족.
**해결**: 빈 상태 카드에 일러스트 + 다음 액션 명확한 CTA 1개 + 예시 1줄 (예: "마키에게 첫 임무 시키기 →").

#### R-8. 로딩 UI 부재 (스켈레톤)
**문제**: 데이터 fetch 중 "불러오는 중..." 텍스트만. 깜빡임 발생.
**해결**: pixel-frame skeleton 컴포넌트 + 보관함/상품/자동화 허브 적용.

#### R-9. 키보드 단축키 부재 (모달 ESC만)
**문제**: 데스크 화면 다수 액션이 마우스 의존. 파워 셀러 효율 ↓
**해결**: 글로벌 단축키 — `⌘K` 빠른 검색, `G→D` 데스크, `G→L` 보관함, `G→P` 상품, `?` 단축키 가이드 모달.

#### R-10. ErrorBoundary 부재
**문제**: 컴포넌트 하나 깨지면 페이지 전체 흰 화면. 운영 시 위험.
**해결**: `app/error.tsx` 글로벌 + `/app`, `/desk` 별 단계별 boundary.

#### R-11. 글로벌 검색 부재
**문제**: 사용자가 "어제 만든 광고 캡션" 검색하려면 보관함 들어가야 함.
**해결**: `⌘K` 트리거 글로벌 검색 (라이브러리 + 상품 + 비서 응답 + 페이지 네비).

#### R-12. AGENTS 데이터 정적 — 사용자 컨텍스트 미반영
**위치**: `data/agents.ts`
**문제**: 모든 사용자가 같은 sample 채팅. 브랜드 컨텍스트 주입 후에도 sample은 일반.
**해결 (Phase 2 범위 외)**: 프롬프트로 동적 sample 생성 — 비싸므로 일단 패스, 기록만.
