## 📌 P-1. PostgreSQL 마이그레이션 — 호스팅 결정

### 왜 필요한가
- SQLite는 단일 노드 파일 DB. 서버리스(Vercel) 배포 시 콜드 스타트마다 새 인스턴스 → WAL 동기화 충돌 + 쓰기 손실 위험
- 다중 인스턴스 + 자동 백업 + PITR(Point-In-Time Recovery)이 필요
- 사용자 트래픽이 늘면 SQLite의 동시 쓰기 락이 병목

### 옵션 비교

| 서비스 | 무료 티어 | 한국 리전 | PITR | 추천도 | 비고 |
|---|---|---|---|---|---|
| **Supabase** | 500MB / 2 프로젝트 | Singapore | Pro부터 | ⭐⭐⭐⭐⭐ | Auth/Storage/Realtime 통합 — Crewmate에 가장 잘 맞음 |
| **Neon** | 500MB / 0.25 vCPU | US/EU | Pro부터 | ⭐⭐⭐⭐ | 콜드 스타트 빠름, branching 지원 |
| **Railway** | $5 크레딧/월 | US/EU | 자체 백업 | ⭐⭐⭐ | 단순함 |
| **AWS RDS** | t4g.micro | Seoul | 35일 | ⭐⭐⭐ | 한국 리전 + 안정성, 셋업 복잡 |

### 추천: **Supabase**
이유:
1. 한국에서 가까운 Singapore 리전 (50~80ms 레이턴시)
2. PostgreSQL + Auth(JWT) + Storage(이미지/파일) 통합 — 향후 사용자 업로드 이미지 기능 시 추가 인프라 불필요
3. Realtime 채널 제공 → 우리 SSE bus를 그대로 교체 가능 (현재 인메모리 → 멀티 인스턴스 대응)
4. Free 500MB → MAU 100~500명 충분, 그 이상은 Pro $25/월

### 작업량 추정
- Prisma 스키마 정의 + 마이그레이션 SQL: **0.5일**
- `lib/db.ts` → Prisma 어댑터 전환: **2~3일** (모든 method 재작성)
- 데이터 이전 (현재 SQLite users.db → 새 Postgres): **0.5일** (3명뿐이라 작음)
- SSE bus → Supabase Realtime 교체: **1일** (선택)

### 사용자 결정 필요
- [ ] **호스팅 선택**: Supabase / Neon / 기타 / 보류
- [ ] **마이그레이션 시점**: 지금 / 결제 오픈 직전 / 사용자 100명 후

## 📌 P-2. 결제 PG 연동 — 가맹 계약

### 왜 필요한가
현재 `app/api/billing/subscribe/route.ts`는 모킹 상태. `process.env.PAYMENTS_PROVIDER !== "live"` 가드로 production 차단됨. 실서비스 오픈 = 이 모킹을 PG 연동으로 교체.

### 옵션 비교

| 서비스 | 카드 수수료 | 무통장 | 정기 결제 | 영수증 | 한국 셀러 적합도 |
|---|---|---|---|---|---|
| **포트원 (구 아임포트)** | 3.3% | ✅ | ✅ | 자동 | ⭐⭐⭐⭐⭐ |
| **토스페이먼츠** | 2.9~3.5% | ✅ | ✅ | 자동 | ⭐⭐⭐⭐ |
| **카카오페이** | 2.9% | ❌ | ✅ | 카카오 | ⭐⭐⭐ |
| **Stripe** | 2.9% + ₩300 | ❌ | ✅ | 영문 | ⭐ (한국 셀러 부적합) |

### 추천: **포트원 + 토스페이먼츠 dual**
이유:
1. 포트원 = 어그리게이터 → 한 번 연동으로 카드/카카오/네이버페이 다 지원
2. 토스페이먼츠 직접 연동 = 백업 (포트원 장애 대비)
3. 한국 셀러 친숙

### 작업량 추정
- 가맹 계약: 1~2주 (사업자등록증 / 통장사본 / 사이트 심사)
- 결제 위젯 통합: **1일**
- 웹훅 엔드포인트: **1일** (payment.completed / failed / refund / dispute)
- 영수증 발급 + 세금계산서: **0.5일**
- Idempotency 키 처리: **0.5일**
- E2E 테스트: **1일**

총 코딩: **4일** (계약 후)

### 사용자 결정 필요
- [ ] **PG 선택**: 포트원 / 토스페이먼츠 / 카카오페이 / 보류
- [ ] **가맹 계약 시작 시점**: 지금 / 호스팅 결정 후 / 사용자 100명 후

## 📌 P-3. 외부 데이터 도구 — 비서 인터넷 검색

### 왜 필요한가
현재 비서가 "오늘의 인스타 트렌드"라며 답하지만 **실제로 인터넷 접근 도구가 없음** → 학습 데이터 기반 일반론. 한국 셀러 입장에선 가치 ↓

### 옵션 비교

| 도구 | 비용 | 한국어 검색 | 사용 사례 |
|---|---|---|---|
| **Tavily** | $0.005/검색 (무료 1000건/월) | ⭐⭐ (영어 우세) | 마키 트렌드 / 애디 광고 사례 |
| **Serper API** | $50/2500건 | ⭐⭐⭐ | 네이버/구글 통합 |
| **네이버 검색광고 API** | 무료 | ⭐⭐⭐⭐⭐ | 애디 키워드 검색량/CPC |
| **Brave Search API** | $3/1000 | ⭐⭐ | 일반 웹 |

### 추천: **Tavily (마키·데일리·애디 공통) + 네이버 검색광고 API (애디 전용)**
이유:
1. Tavily 무료 티어로 시작 → MAU 100명 시 한 달 ~₩50K
2. 네이버 검색광고는 한국 셀러 광고 운영의 핵심 — API 무료, 사업자번호만 있으면 발급
3. fetch_url 도구는 Anthropic Claude API에 이미 web_fetch가 있을 수도 있어 별도 도구 불필요할 수 있음

### 작업량 추정
- Tavily API 연동 (`lib/claude/tools.ts`에 `web_search` 도구 추가 + runner에서 실행): **0.5일**
- 네이버 검색광고 API 연동: **1일** (HMAC 인증 + 5개 엔드포인트)
- 비서 시스템 프롬프트에 "사용 가능 도구" 안내 추가: **0.5일**
- 비용 추적 + 한도 강제: **0.5일**

### 사용자 결정 필요
- [ ] **Tavily 도입**: 예 / 아니오 / 보류
- [ ] **네이버 검색광고 API 발급**: 사용자가 신청 (사업자번호 필요)
- [ ] **검색 비용 추가 한도 정책**: 무료 플랜에서 검색 차단 / 횟수만 차감

## 📌 P-5. 봇 회원가입 추가 방어 — Cloudflare Turnstile

### 왜 필요한가
현재 honeypot + 폼 노출 시간 검증으로 90%는 막힘. 그러나 정교한 봇은 우회 가능. 결제 시작 후 스팸 가입으로 AI 호출 abuse 위험.

### 옵션 비교

| 서비스 | 비용 | 사용자 마찰 | 차단율 |
|---|---|---|---|
| **Cloudflare Turnstile** | 무료 | 없음 (invisible) | ⭐⭐⭐⭐⭐ |
| **hCaptcha** | 무료 | 가끔 클릭 | ⭐⭐⭐⭐ |
| **reCAPTCHA v3** | 무료 (1M/월) | 점수 기반 | ⭐⭐⭐ |

### 추천: **Cloudflare Turnstile**
- 무료, 사용자에게 보이지 않음, Cloudflare 인프라 → 한국 도착 빠름
- 가입은 1분 작업

### 작업량 추정
- Turnstile 사이트 등록 (사용자): 5분
- 회원가입 폼에 Turnstile 위젯 추가: **0.5일**
- 서버에서 토큰 검증 (`/api/auth/signup` 진입 직후): **0.5일**

### 사용자 결정 필요
- [ ] **Turnstile 사이트 키 발급**: 사용자가 https://dash.cloudflare.com/?to=/:account/turnstile 에서 발급
- [ ] **도메인**: 운영 도메인 입력 필요
